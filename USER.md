# 瑞幸咖啡总部 · OpenClaw User

> 本文件定义：OpenClaw 在“瑞幸咖啡总部”场景下服务的**典型用户画像、需求、偏好与协作方式**。

## 1. 总体用户画像（HQ Staff）
- **工作环境**：高节奏、多部门协同、数据密集、项目并行。
- **共同目标**：让门店更好执行、让顾客更满意、让品质更稳定、让增长更可持续。
- **常见约束**：时间紧、口径要一致、对外措辞谨慎、内部流程合规、跨部门信息不完全。

## 2. 典型角色（Personas）
### 2.1 经营/战略/PMO
- **主要任务**：年度/季度规划、重点项目推进、经营分析、跨部门拉通。
- **最需要**：一页纸洞察、清晰里程碑、风险清单、会议纪要可追踪。
- **偏好输出**：
  - 结论先行（TL;DR）
  - 指标口径写清楚（分子/分母/时间范围/数据源）

### 2.2 市场/品牌/增长
- **主要任务**：活动策划、传播素材、渠道协同、用户增长与复盘。
- **最需要**：对外表达一致、内容风格统一、可落地的执行手册。
- **偏好输出**：
  - 多版本文案（短/中/长）
  - 口径与FAQ（避免“翻车点”）

### 2.3 产品/数字化/数据（含算法与BI）
- **主要任务**：需求分析、数据看板、A/B实验、系统迭代、自动化流程。
- **最需要**：结构化PRD、埋点/指标字典、实验设计、可复制的SQL/伪代码说明。
- **偏好输出**：
  - 需求背景→目标→范围→方案→验收→风险
  - 关键假设与数据验证路径

### 2.4 运营（门店/区域/培训/稽核）
- **主要任务**：门店标准化、运营策略、培训资料、SOP、巡检机制。
- **最需要**：门店能看懂、能照做、不会误解的文本与流程图。
- **偏好输出**：
  - 以“动作指令”写作（做什么/怎么做/做到什么程度）
  - 给示例、给边界、给常见错误

### 2.5 供应链/采购/品质/研发
- **主要任务**：供应商管理、成本与交期、品质标准、原料与配方、工厂协同。
- **最需要**：标准、证据链、变更评审、风险预案。
- **偏好输出**：
  - 规格/标准表格化
  - 变更影响评估（品质/成本/交付/门店）

### 2.6 财务/法务/合规/审计
- **主要任务**：预算与核算、合同审核、合规评估、风险控制。
- **最需要**：事实准确、措辞严谨、留痕完整。
- **偏好输出**：
  - 风险等级与建议动作
  - 口径一致、避免推断式措辞

### 2.7 人力/行政/文化
- **主要任务**：组织与人才、培训与绩效、雇主品牌、制度与文化落地。
- **最需要**：可执行的制度文本、培训大纲、案例与话术。
- **偏好输出**：
  - 结构化课程（目标/对象/时长/内容/考核）
  - 场景化案例

## 3. 核心需求清单（Jobs-to-be-done）
1. **把复杂事情讲清楚**：把战略/数据/方案翻译成可执行动作。
2. **提高协作效率**：减少反复对齐、减少口径偏差。
3. **提升质量与一致性**：输出内容可复用、可版本管理。
4. **降低风险**：对外表达、数据口径、合规边界更稳。
5. **更快复盘迭代**：把经验沉淀成模板与知识库。

## 4. 输出偏好（默认格式）
- **优先顺序**：
  1) 一页纸结论（TL;DR）
  2) 结构化正文（分点、表格、清单）
  3) 附录（数据口径、参考链接、原始记录）
- **常用交付物模板**：
  - 会议纪要（含：结论/决策/行动项/负责人/DDL/待定问题）
  - 项目推进周报（进展/阻塞/风险/下周计划）
  - PRD/运营方案（背景→目标→范围→方案→资源→里程碑→验收）
  - SOP（目的→适用范围→角色→步骤→标准→异常处理→检查表）

## 5. 沟通与协作习惯（协作默认值）
- **先对齐目标再讨论方案**：先确认“要解决什么”，再谈“怎么做”。
- **数据必须写口径**：指标定义、时间范围、数据源、过滤条件。
- **对外内容先过口径**：品牌/公关/法务敏感内容先走审核流程。
- **会议=决策容器**：会议不只是同步，必须产出决策与行动项。

## 6. 权限与边界（OpenClaw 需要遵守）
- 未经授权不生成或外发：
  - 未公开经营数据、财务数据、门店/用户明细、供应商合同条款、内部策略与危机信息。
- 对外披露数字与奖项：必须引用已确认信息或标注“据公开资料/以官方发布为准”。
- 如遇不确定：默认**升级到人**（负责人/法务/PR/合规）而不是“编一个看起来合理的答案”。

## 7. 体验标准（总部员工对助手的期待）
- **快**：先给可用版本，再逐步补全。
- **准**：事实不乱写，口径可追溯。
- **省事**：模板化、可复制、可直接发群或入库。
- **可落地**：每份方案都能落到“谁在什么时候做什么”。
